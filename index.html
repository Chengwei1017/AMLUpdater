<h1 id="amlupdater">AMLUpdater</h1>

<h2 id="a-program-used-for-update-wrf-land-use-data-with-high-resolution-land-use-data-by-airmonster-team-from-chengdu-academy-of-environmental-sciences">A program used for update wrf land use data with high resolution land use data, by Airmonster team from Chengdu Academy of Environmental Sciences.</h2>

<p>The code was written in R. geo_em.d* data was processed off-line which means you should copy your data from WPS dir and then process with this program. Tests were carried out on Window with R 3.5.2.</p>

<p>This program was inspired by this link: http://bbs.06climate.com/forum.php?mod=viewthread&amp;tid=2435</p>

<p>The default land use data in this program was FROM-GLC shared by Tsinghua University: http://data.ess.tsinghua.edu.cn/fromglc2015_v1.html</p>

<h2 id="how">HOW</h2>
<p>As you know, land use was an important part of WRF static data, and the default land use data from the model were outdated for most cities in China because of the rapid urbanization. It is quite difficult for beginners to update the land use data which acquires technics including programming and gis, so we provide an easy way to start with.
FROM-GLC 2015 v1 data with remapping according to modis land use scheme in WRF was used in this program to calculate the ratio of different land use types, and updates were made to geo_em data.</p>
<ul>
<li>Dependency of AMLUpdater are a list of R packages including:<br />
<em>Raster</em> : raster process, including clip and merge.<br />
<em>RNetCDF</em> : RW for netcdf files.<br />
<em>doParallel</em> ：Parallel process for grids to improve effiency.</li>
<li>Before running the program, some parameters should be modified:<br />
<em>DATDIR</em> :Local path for FROM-GLC data.<br />
<em>GEODIR</em> :Upper folder for geo_em files.<br />
<em>case</em> :Under GEODIR, and inside which are geo_em.d0?.nc<br />
<em>NSL</em> :LAT range in format of (start-lat, end-lat, step), notice step should be 10 for FROM-GLC 2015v1.<br />
<em>EWL</em> : LON range in format of (start-lon, end-lon, step), notice step should be 10 for FROM-GLC 2015v1.<br />
<em>DLD</em> : T/F, whether data needs to be downloaded, T for the first time.<br />
<em>MAXDM</em> : max number of the nested domain in WRF.</li>
</ul>

<h2 id="tips">TIPS</h2>
<p>Inside the code, a special parameter named ubonly should be set to T if you want to update urban and waterbody only, sometimes it is a better choice for you.
In our tests, after updating the land use data, correlation of temperature were slightly improved for all 13 observation points in Chengdu, and correlation of the wind speed of 5 observation points were improved.</p>
